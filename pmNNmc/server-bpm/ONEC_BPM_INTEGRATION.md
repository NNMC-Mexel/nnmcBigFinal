# BPM и 1С

## Контур обмена

Обмен однонаправленный: BPM отправляет запросы в HTTP-сервис расширения 1С, а 1С возвращает результат обработки. Основная конфигурация 1С не изменяется.

Для документов используется:

```text
POST {ONEC_BPM_REQUEST_URL}?Command=Check&Type={1..17}
POST {ONEC_BPM_REQUEST_URL}?Command=Create&Type={1..17}
```

Для справочников используется отдельное пространство типов:

```text
POST {ONEC_REFERENCE_REQUEST_URL}?Command=Check&Type={1..5}
```

Авторизация Basic. Тело запроса JSON. Перед отправкой кадрового документа BPM проверяет физическое лицо командой `Check`, `Type=1`.

Коды ответа 1С:

- `1` — операция выполнена;
- `2` — ошибка авторизации или обработки;
- `3` — объект отсутствует при проверке.

## Переменные окружения

```env
ONEC_BPM_REQUEST_URL=http://192.168.40.83/copy10062025-rest/hs/InComingRequest/Request
ONEC_REFERENCE_REQUEST_URL=http://192.168.40.83/copy10062025-rest/hs/InComingRequest/RequestInfo
ONEC_VACATION_BALANCE_URL=http://192.168.40.83/copy10062025-rest/hs/InComingRequest/Request
```

Пути являются отдельными переменными, потому что существующий `ONEC_API_URL` уже используется endpoint-ами `NNMC` для карточек сотрудников и табелей.
